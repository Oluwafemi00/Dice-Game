## Overview

Dicee is a two-player (player vs computer) dice game played to 10 points. Every aspect of the game — dice faces, sound effects, win/loss fanfares — is generated programmatically at runtime. The UI reacts to each roll with animated feedback, outcome-coloured dice faces, and contextual audio.

---

## Features

| Feature                    | Details                                                        |
| -------------------------- | -------------------------------------------------------------- |
| **Player vs Computer**     | First to 10 points wins the game                               |
| **SVG Dice Faces**         | All 6 dice faces rendered as inline SVG — no images            |
| **Outcome Colouring**      | Winning die turns green, losing die turns red, draws turn gold |
| **3D Roll Animation**      | CSS `rotateX/rotateY` keyframe animation on each roll          |
| **Synthesized Sound**      | Dice rattle, win fanfare, and loss jingle via Web Audio API    |
| **Circular Progress Ring** | SVG ring shows each player's score as a % of the win target    |
| **Score Pop Animation**    | Score number bounces on update                                 |
| **Streak Tracking**        | Consecutive wins tracked and displayed with 🔥 emoji           |
| **Game History**           | Last 15 round outcomes as coloured dot indicators              |
| **Result Badges**          | "▲ WINS" / "DRAW" badges appear under each die                 |
| **Status Panel**           | Win/lose/draw state shown with contextual colour               |
| **Reset**                  | Full game reset returns all state to zero                      |
| **Felt Texture**           | CSS diagonal grid gives the background a casino felt feel      |

## Technical Highlights

### SVG Dice

All 6 dice faces are generated by a `getDiceSVGContent(n, outcome)` function that:

1. Selects background and dot colour based on win/lose/draw outcome
2. Maps each face number to an array of `[cx, cy]` dot coordinates
3. Returns an SVG string injected via `innerHTML`

### Web Audio Synthesis

- **Dice rattle** — 4 noise bursts with staggered `setTimeout` calls and random bandpass frequencies
- **Win fanfare** — ascending major triad (C5, E5, G5) with 110ms stagger
- **Loss jingle** — descending minor phrase (G4, E4, C4) with sawtooth oscillator

### Other Highlights

- **`AudioContext` resumed on first `pointerdown`** — satisfies browser autoplay policy
- **DOM reflow trick** (`void el.offsetWidth`) restarts CSS animations cleanly
- **Circular ring** uses `2πr` circumference math with `stroke-dashoffset` for clean progress mapping
- **Playfair Display + DM Sans** — serif for the game title and dice numbers, sans for UI labels

## Design Decisions

- **Deep felt green** (`#0A1F12`) base — unmistakable casino table reference without being literal
- **Gold accent** (`#C9A84C`) for scores and status — conveys prestige and stakes
- **Outcome-coloured dice** — no need to read text; the dice colour tells you who won instantly
- **History dots** — 15-entry rolling record gives players context on the session's momentum
